# CS320
This repository includes work I completed in CS 320: Software Test Automation QA. The projects shows my experience writing Java classes, creating Junit tests, and using testing to verify that software meets its requirements.
## Reflection
## How can I ensure that my code, program, or software is functional and secure?
One thing I learned in this course is that testing should happen throughout development, not just after everything is finished. Writing unit tests helped me verify that each class worked the way it was supposed to. I also learned that validating data and following the project requiremenents an prevent problems before they become bugs or security issues.
## How do I interpret user needs and incorporate them into a program?
I usually start by reading through the requirements and figuring out what the program s expected to do. Once I understand the goal, I break it into smaller tasks and build one feature at a time. As I finish each part, I use testing to make sure it matches what the user asked for before moving on.
## How do I approach designing software?
When I design software, I try to keep things simple and organized. I like separating different responsibilities into their own classes because it makes the code easier to read, test, and update later. This course showed me that spending time planning the design upfront makes development and debugging much easier in the long run.
